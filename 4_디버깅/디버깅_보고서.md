## 🐛 발견한 오류 #1

### 1) 어떤 문제가 발생했나요?
프로그램을 실행하면 컴퓨터가 말하는 숫자가 4개가 되는 경우가 있었어요.
1~3개만 말해야 하는데 규칙 위반이 발생함.

### 2) 오류가 발생한 코드
com = 4 - (n % 4);
// 여기에서 결과가 4가 나오는 경우가 있음


### 3) 왜 오류가 생겼나요?
현재 숫자 n이 이미 4의 배수일 때
4 - (n % 4) 결과가 4가 되어서 생긴 문제임

### 4) 어떻게 고쳤나요?
com = 4 - (n % 4);
if (com == 4) com = 1;   // 4 나오면 1로 바꿔줌


### 5) 고친 후 결과
 ✅ 잘 작동해요!

## 🐛 발견한 오류 #2

### 1) 어떤 문제가 발생했나요?
사용자가 1~3이 아닌 숫자를 입력하면 프로그램이 바로 다음 단계로 넘어가 버림

### 2) 오류가 발생한 코드
scanf("%d", &me);
// 입력 검사 없이 바로 진행됨


### 3) 왜 오류가 생겼나요?
```
사용자가 0이나 10 같은 잘못된 숫자를 입력해도
검사 코드가 없어서 그대로 다음 단계로 넘어갔기 때문입니다.
```

### 4) 어떻게 고쳤나요?
```if (me < 1 || me > 3) {
    printf("1~3 중에 고르라고!\n");
    continue;    // 다시 입력 받게 함
}

```

### 5) 고친 후 결과
- [ ] ✅ 잘 작동해요!
